# Inkhaven 1.2.7 — in flight

The 1.2.7 cycle is the polish pass on top of the
timeline-heavy 1.2.6 release. Themes are scoped narrow on
purpose: real-project bug reports surfacing in the first
weeks after a major feature ship, plus the small follow-ups
listed at the end of the 1.2.6 release notes. The structural
work (full-screen split-view ergonomics + F4 rework) is
parked for 1.3.

This document is a living scratchpad — entries land here as
the work lands. Each item starts as a one-line plan; gets
fleshed out when the corresponding feature ships; gets a
chord / config / hook callout when relevant.

## Planned themes

### Timeline polish

- **Project-overlay book prefix.** Events from different
  books in the `Ctrl+V Shift+T p` overlay currently render
  with bare track names; collisions on a track name across
  books are invisible. Add the book slug as a prefix so
  cross-book tracks read as `aerin-saga/main` vs
  `khaal-arc/main` rather than two rows called `main`.
- **`inkhaven export timeline` (SVG / PNG / Typst).** The
  swim-lane view + the event picker are TUI-only today.
  An export subcommand renders the same view to a file so
  authors can attach the timeline to a query letter, a
  pitch doc, or a private wiki page. Format flags mirror
  `inkhaven export pdf` style.
- **Cascade-impact AI when an event is moved.** Edit-event
  prompt (`Ctrl+V Shift+I`) currently rewrites timing
  without flagging that 6 linked paragraphs now refer to a
  date in the past, that a season-precision event now
  overlaps a war it didn't before, etc. New chord (or
  optional opt-in on the existing Shift+I commit) sends
  the before/after state + the linked paragraphs to the
  AI for a cascade audit.
- **Scrivener `<CustomMeta>` date import bridge.** The
  Scrivener importer (1.2.4) ignores `<CustomMeta>` fields
  even when they carry obvious dates. Map known field
  names (`Date`, `Story Date`, `Event Date`, …) onto
  events under a per-imported-book Timeline chapter.

### AI polish

- **F11-on-paragraph "suggest event" variant.** When the
  cursor sits inside prose that names a date or a clearly
  scene-shaped moment, F11 (currently the explain-diag
  chord on Linux/Windows; Ctrl+F12 on macOS) gains a
  paragraph mode that proposes a draft event from the
  prose. User accepts → event lands in the book's
  Timeline chapter, pre-linked to the source paragraph.
