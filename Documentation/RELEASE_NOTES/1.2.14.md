# Inkhaven 1.2.14 — in flight

1.2.13 was the **Language book** release — a
first-class invented-language workbench with
five-chapter sub-books (Meta / Dictionary /
Grammar / Phonology / Sample texts), HJSON-
fronted entries, lexicon overlay + status chip
in the manuscript editor, AI translation flow
with `Ctrl+B Q` / `Ctrl+B Shift+Q`, full CLI
surface (init, add-word, --import CSV with --new
/ --force, remove-word, list, doctor with --json,
export), and two tutorials (49 + 50) with a
worked Tira sample CSV.

This document is a living scratchpad — entries
land here as the work lands.  Each item starts
as a one-line plan; gets fleshed out when the
corresponding feature ships; gets a chord /
config / hook callout when relevant.

## Planned themes

### Carryovers from 1.2.13

#### Language book — Phase D.2 (deferred from 1.2.13)

  * **`--format grammar` export** — printable
    grammar reference (TOC + chapter per rule
    category + examples tables + cross-refs).
    The current grammar HJSON template carries
    the right shape; the exporter needs to
    parse + format it.
  * **`--format phrasebook` export** — Sample-
    text chapter rendered as a phrasebook,
    gloss on left, working language on right.
  * **`--format csv` export** — round-trip-
    compatible CSV that the `--import` path
    can re-ingest.  Closes the import/export
    loop.
  * **`inkhaven language test <name>`** —
    headless roundtrip translation drift CLI.
    Walks a corpus of sample texts through
    `Ctrl+B Q` → `Ctrl+B Shift+Q` and reports
    drift.  Catches grammar-rule inconsistencies
    before they bite in the manuscript.  The
    in-TUI version of this workflow already
    works; this CLI version is for CI gates.
  * **`inkhaven language translate <language>
    <text>`** — headless flavour of the in-TUI
    `Ctrl+B Q` chord.  Useful for shell-pipe-
    driven authoring.
  * **`inkhaven language define-rule <language>
    <rule_id> --category <cat>`** — open the
    rule template in `$EDITOR` for hand-editing
    the HJSON.  Pairs with the deferred
    `--format grammar` exporter.

#### Language book — UX polish

  * **`Ctrl+B Shift+R` reverse-lookup picker
    in Language book** — find dictionary entry
    by translation.  The `LanguageEntryIndex`
    from 1.2.13 already keys by surface form;
    just needs an inverse `translation → entry`
    index + a picker modal mirroring `LlmPicker`.
  * **`Ctrl+B Shift+W` word-of-the-day card**
    in the manuscript editor — random Dictionary
    entry as a floating card.  Pairs with the
    `Ctrl+B Shift+W` phonotactic generator in
    the Language book (generate a candidate
    word conforming to the language's
    phonotactics).
  * **Card renderers** for Dictionary / Grammar
    / Phonology paragraphs viewed inside the
    Language book — the §7 / §10 visualisations
    from the proposal.  Significant render work.
  * **Inline gloss-table renderer** — the AI
    pane currently renders the LLM's markdown
    table as-is.  A dedicated renderer that
    aligns the `source · gloss · target`
    columns with column-typed colours would
    make the gloss easier to scan during
    proofreading.
  * **Confidence-flag chip** — when the LLM
    returns "dictionary missing for X", a
    one-key chord to jump straight into
    `inkhaven language add-word`'s seed prompt
    would close the diagnose → fix loop.
  * **Inflection-aware status-bar chip** —
    when the cursor lands on an inflected form
    (not the lemma), surface the paradigm name
    alongside the lemma + translation:
    `[atatal (plural of atal) · noun · river]`.

### Carryovers from 1.2.12

  * **Mouse-driven scene-break selection** —
    deferred from 1.2.11 → 1.2.12 → 1.2.13;
    still on the list.  Drag from one
    scene-break line to the next selects the
    scene.  Pairs with the existing
    `Ctrl+B <` / `Ctrl+B >` scene-break
    navigation.

  * **Inline streaming-response pane inside
    the `Ctrl+B 0` HJSON editor.**  The 1.2.12
    LLM review (`Ctrl+R`) routes the response
    into `App.inference` — visible in the AI
    pane only after closing the modal.  Carving
    the bottom 1/3 of the HJSON-editor modal
    for an inline response pane would let the
    user critique + fix without context-
    switching.

  * **Full per-overlay parity for the split-
    view secondary editor.**  Search highlights,
    style-warning detectors, lexicon highlights,
    grammar overlays, tag overlays all run on
    the primary's `draw_editor` but not on
    `draw_secondary_editor`.  In practice OK
    for translation / reference workflows but a
    future commit could thread the overlay
    passes through.

## Implementation log

### Threads system book — Phase A.1 (foundation)

Lays the groundwork for the plot-intelligence
tooling described in
`Documentation/PROPOSALS/1.2.14_PLAN.md`.  Phase
A.1 is the storage + scaffold layer only — no
weave view (A.2), no AI thread audit (A.3).

* **`SYSTEM_TAG_THREADS = "threads"`** added to
  `src/store/mod.rs`.  Top-level `Threads` system
  book auto-creates on next project open (or
  upgrade from a pre-1.2.14 project) alongside the
  existing system books.  Lands at canonical order
  index 6 — after Artefacts, before Language — so
  the worldbuilding cluster (Places / Characters /
  Artefacts / Threads / Language) groups together
  in the tree.  Existing user books shift up by 1
  if they collided with the slot; the heal-pass
  at the end of `ensure_system_books` mops up.

* **`inkhaven thread add <name>`** CLI subcommand.
  Creates a Paragraph under the Threads system
  book, seeds the body with the full commented
  HJSON template per proposal §3.1 (identity:
  `title` / `status` / `weight` pre-filled from
  the CLI flags; arc shape: `opening` /
  `midpoint` / `payoff`; connections:
  `characters` / `places` / `artefacts` /
  `related_threads`; metadata: `tension` /
  `register` / `notes`).  Sets `content_type =
  "hjson"` so the editor renders with syntax
  highlighting + the paragraph status bar shows
  `[hjson]`.  Disk-write + bdslib-write follow
  the canonical two-step pattern from Language
  entries.  Rejects duplicates by case-insensitive
  title.

  ```
  $ inkhaven thread add "inheritance subplot" \
      --status develop --weight subplot
  added thread `inheritance subplot` to Threads (develop · subplot)
    open Threads/inheritance subplot in the editor to fill opening / midpoint / payoff
  ```

* **`inkhaven thread list`** CLI subcommand.
  Walks the Threads subtree, parses each entry's
  HJSON summary fields, prints a column-aligned
  table:

  ```
  $ inkhaven thread list --status develop
    name                  status    weight  tension   ch  pl  art
    ----------------------------------------------------------
    inheritance subplot   develop  subplot        7    2   1    0
    redemption arc        develop    major        5    1   0    1
  ```

  `--status` / `--weight` flags filter case-
  insensitively.

* **Tree-pane `+` chord under Threads** —
  `commit_add` now detects when the parent's
  ancestor chain includes the Threads system book
  and seeds the body with the thread HJSON
  template (`title` pre-filled from the typed
  name; `status: "setup"`, `weight: "major"`).
  `content_type` flips to `"hjson"`.  Same
  template-seed pattern Language Grammar/Phonology
  paragraphs use, just with a different schema.
  No bucket-routing needed — threads are flat
  Paragraphs under the system book (or chapters
  the author adds for grouping).

* **Phase A.1 tests** — 7 new:
  `seed_thread_body_includes_core_fields`,
  `seed_thread_body_uses_name_when_title_omitted`,
  `seed_thread_body_prefers_explicit_title`,
  `seed_thread_body_parses_as_valid_hjson`,
  `parse_thread_summary_extracts_filter_fields`,
  `parse_thread_summary_returns_none_on_empty`,
  `escape_hjson_handles_quotes_and_backslashes`.

572 tests pass (was 565 — +7).  Zero new
dependencies.  Phase A.2 (thread weave view +
`Ctrl+V Shift+H` picker) and Phase A.3 (AI thread
audit + `Ctrl+V Shift+A`) are queued.
