# Inkhaven 1.2.15 — in flight

1.2.14 was the **plot intelligence + inline
comments + quick wins** release — a new `Threads`
system book with swim-lane weave view (`Ctrl+V
Shift+H`), AI thread audit (`Shift+A`), and
deterministic thread doctor (`Shift+D`); inline
sidecar comments with span anchoring + project-
wide panel + AI digest (`Ctrl+V c` / `Shift+C`);
14 init templates including 5 Russian and 3
international new (epic-fantasy / mystery /
french-novel); HJSON snippet expansion with 8
placeholders incl. `{cursor}` split-paste; AI
continuation drafting (`Ctrl+V d`); inline
footnote insertion (`Ctrl+V f`); project word-
count goal modal with 30-day projection
(`Ctrl+V Shift+G`); style-transfer rewrite
(`Ctrl+V y`).  Three new CLI surfaces: `thread`,
`comments`, `template`.  Tests 565 → 637 (+72).

This document is a living scratchpad — entries
land here as the work lands.  Each item starts
as a one-line plan; gets fleshed out when the
corresponding feature ships; gets a chord /
config / hook callout when relevant.

## Planned themes

### Carryovers from 1.2.14

#### Snippet expansion — picker placeholders + bund: prefix

The Q.2 snippet system ships with 8 built-in
placeholders that resolve synchronously inside
`expand_placeholders`.  Three placeholders + one
prefix were deferred because they need to pause
the synchronous expansion pipeline for an async
event:

  * **`{char_lookup}` / `{place_lookup}` /
    `{artefact_lookup}`** — open the corresponding
    system-book picker and substitute the chosen
    entry's display name (or wiki-link target,
    depending on the surrounding text).  Needs a
    pending-snippet state machine: the editor
    stashes the partial expansion, opens the picker
    modal, and resumes the substitution on commit.
  * **`bund:` prefix** — when the snippet body
    starts with `bund:`, the rest is interpreted
    as a Bund-VM program whose output (top of stack
    as string) becomes the expansion.  Needs a Bund
    interpreter handle on the snippet expansion
    path plus error handling for non-stringable
    results.  Pairs naturally with the existing
    `ink.*` stdlib (`ink.fs.read_file`,
    `ink.editor.*`).

  Both unlock advanced authoring flows but require
  the same underlying mechanism — once the async
  state machine lands, all four arrive together.

#### Language-book D.2 carryovers

Carried from 1.2.13 into 1.2.14 and still queued.
Mostly CLI exports + headless workflow tools:

  * **`inkhaven language export --format grammar`**
    — printable grammar reference (TOC + chapter
    per rule category + examples tables + cross-
    refs).  Schema is already there; the exporter
    needs to parse + format.
  * **`inkhaven language export --format phrasebook`**
    — Sample-text chapter rendered as a phrasebook
    (gloss on left, working language on right).
  * **`inkhaven language export --format csv`** —
    round-trip-compatible CSV that the `--import`
    path can re-ingest.  Closes the import/export
    loop.
  * **`inkhaven language test <name>`** — headless
    roundtrip-translation drift CLI.  Walks a
    corpus of sample texts through `Ctrl+B Q` →
    `Ctrl+B Shift+Q` and reports drift.  Catches
    grammar-rule inconsistencies before they bite
    in the manuscript.  Pairs with CI gates.
  * **`inkhaven language translate <language>
    <text>`** — headless flavour of the in-TUI
    `Ctrl+B Q` chord.  Shell-pipe-friendly.
  * **`inkhaven language define-rule <language>
    <rule_id> --category <cat>`** — opens the rule
    template in `$EDITOR` for hand-editing the
    HJSON.  Pairs with the deferred `--format
    grammar` exporter.

#### Language-book UX polish (from 1.2.13)

  * **`Ctrl+B Shift+R` reverse-lookup picker** in
    Language book — find dictionary entry by
    translation.  `LanguageEntryIndex` already
    keys by surface form; needs an inverse
    `translation → entry` index + a picker modal
    mirroring `LlmPicker`.
  * **`Ctrl+B Shift+W` word-of-the-day card** in
    the manuscript editor — random Dictionary
    entry as a floating card.  Pairs with the
    `Ctrl+B Shift+W` phonotactic generator inside
    the Language book.
  * **Card renderers** for Dictionary / Grammar /
    Phonology paragraphs viewed inside the Language
    book — the §7 / §10 visualisations from the
    proposal.  Significant render work.
  * **Inline gloss-table renderer** — the AI pane
    currently renders the LLM's markdown table
    as-is.  A dedicated renderer that aligns
    `source · gloss · target` columns with column-
    typed colours would make the gloss easier to
    scan during proofreading.
  * **Confidence-flag chip** — when the LLM
    returns "dictionary missing for X", a one-key
    chord to jump straight into `inkhaven language
    add-word`'s seed prompt would close the
    diagnose → fix loop.
  * **Inflection-aware status-bar chip** — when
    the cursor lands on an inflected form (not the
    lemma), surface the paradigm name alongside
    the lemma + translation:
    `[atatal (plural of atal) · noun · river]`.

### Carryovers from 1.2.12

  * **Mouse-driven scene-break selection** —
    deferred from 1.2.11 → 1.2.12 → 1.2.13 → 1.2.14;
    still on the list.  Drag from one scene-break
    line to the next selects the scene.  Pairs
    with the existing `Ctrl+B <` / `Ctrl+B >`
    scene-break navigation.
  * **Inline streaming-response pane inside the
    `Ctrl+B 0` HJSON editor.**  The 1.2.12 LLM
    review (`Ctrl+R`) routes the response into
    `App.inference` — visible in the AI pane only
    after closing the modal.  Carving the bottom
    1/3 of the HJSON-editor modal for an inline
    response pane would let the user critique +
    fix without context-switching.
  * **Full per-overlay parity for the split-view
    secondary editor.**  Search highlights, style-
    warning detectors, lexicon highlights, grammar
    overlays, tag overlays all run on the primary's
    `draw_editor` but not on `draw_secondary_editor`.
    In practice OK for translation / reference
    workflows but a future commit could thread the
    overlay passes through.

### Other candidates

  * **Sixth language for the supported set** —
    Italian / Portuguese / Japanese.  Substantial
    content lift: stop-words + Snowball stemmer
    plumbing (where available) + SDT vocabulary
    lists + embedded prompt set.  Bootstrap CLI
    already exists.

  * **More "diagnose → AI fix" pairs.**  The
    1.2.11 rhythm-rewrite chord (`Ctrl+B Shift+M`)
    pairs with the rhythm gauge (`Ctrl+B Shift+H`);
    same shape could apply to repeated-phrase
    detector → AI rewrite, weak-verb detector →
    AI replace, etc.

  * **Live theme preview in config TUI.**  The HSL
    slider shipped in 1.2.11 but its preview is
    widget-local.  Applying the candidate colour
    to the live editor pane would catch contrast
    issues without saving + restarting.

  * **GitHub Release object follow-up.**  1.2.11,
    1.2.12, 1.2.13 tags were never pushed at all;
    1.2.14 has both a tag AND a pushed remote, but
    no GitHub Release UI object.  Backfill the
    missing tags (cherry-pick from each release
    commit) and create Release objects for all
    four, then either keep refreshing the gh CLI
    `workflow` scope before each release or
    automate via `.github/workflows/release.yml`.
