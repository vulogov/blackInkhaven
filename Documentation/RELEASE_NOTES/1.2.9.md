- **Rename "wiki-links" → "paragraph links" / "linked
  paragraphs" everywhere.**  The 1.2.6 / 1.2.7 docs and a
  handful of in-code comments call `Node.linked_paragraphs`

  `Ctrl+V Shift+L`) + Bund (`ink.event.link_paragraph`).

  be one of the first commits in the cycle so later
  features can call them by the right name from day one.

## Planned features

### TTS read-aloud (with Russian support)

Reading prose aloud catches awkward phrasing that the
eye glosses over.  Wire a configurable TTS engine
behind `Ctrl+B R`:

  Ctrl+B R   — read the current paragraph; second press stops
  status     — "TTS: paragraph X of Y · Esc to stop"

HJSON config shape (under `editor.tts`):

```hjson
editor: {
  tts: {
    enabled: false           // default off — no surprise audio
    engine: "say"            // "say" | "piper" | "espeak-ng" | "shell"
    voice: "Milena"          // engine-specific
    rate: 180                // words per minute (say only)
    shell_command: ""        // when engine = "shell": template like
                             // "piper --model {voice} | aplay -q"
  }
}
```

Engine support, ranked by quality × setup-effort:

| Engine        | Russian quality | Setup                                                                   |
| ------------- | --------------- | ----------------------------------------------------------------------- |
| `say`         | decent          | macOS-native; Milena / Yuri / Katya voices via System Settings download |
| `piper`       | best local      | `brew install piper-tts` + ru voice (`ru_RU-irina-medium.onnx`, ~60 MB) |
| `espeak-ng`   | robotic         | `brew install espeak-ng` / `apt install espeak-ng` — universal          |
| `shell`       | engine-specific | HJSON template (escape hatch for ElevenLabs / OpenAI / custom setups)   |

Macos `Milena` + `say` is the recommended default for
Russian — no extra install required after the one-time
voice download.  `piper` is the upgrade path for users
who want neural-quality offline playback.

Out of scope for the first cut: highlighting the
currently-spoken sentence in the editor (would need
phoneme-level callbacks the subprocess interface
doesn't expose).  Sentence-level progress on the
status bar is enough for first ship.

## What's landed so far in 1.2.9

### Ctrl+B W rebranded as "focus mode"

Re-research from the brainstorm caught a smaller issue
than the scratchpad's original framing — `Ctrl+B W` IS
already distraction-free mode (hides Tree / AI / Search /
AI-prompt, forces editor focus), but the codename
"typewriter mode" buries this fact from new users, and
the chord was **completely undocumented** in
KEYBINDING.md.

Changes:

  - `Action::ToggleTypewriter` short label
    "typewriter" → "focus mode".
  - Long help re-written to call this
    "distraction-free / focus mode" explicitly, with a
    one-line note that the HJSON serde key
    (`global.toggle_typewriter`) stays for backward
    compatibility.
  - Status-bar strings ("typewriter mode · Ctrl+B W to
    exit" / "typewriter mode off") rewritten to use
    "focus mode".
  - New row in `KEYBINDING.md` table — previously missing
    entirely.
  - Tutorial 03 (the editor) gets a "Focus mode
    (distraction-free)" section between "The visual
    overlays" and "Find and replace (regex)".

Not changed (intentional):

  - `Action::ToggleTypewriter` enum variant name —
    public via `keys.bindings`; renaming would break
    user HJSON overrides.
  - Serde key `global.toggle_typewriter` — same reason.
  - Internal field `App.typewriter_mode` + the
    `toggle_typewriter_mode()` method — cosmetic; the
    legacy name doesn't surface to users anywhere
    after this commit.

### "Wiki-links" terminology pass

Renamed across active docs + source code.  The chord
set (`Ctrl+V A` / `I` / `L` / `K`), the `linked_paragraphs`
metadata field, and the file `Documentation/Tutorials/19-wiki-links.md`
are all unchanged (the last one keeps its filename so
external bookmarks don't break — its H1 title is the
new "Paragraph links and backlinks" with a disambiguation
note up top).

Scope intentionally limited:

  - Active documentation: KEYBINDING.md, CONFIGURATION.md,
    KEYS_REASSIGNMENT.md, Documentation/README.md,
    Tutorials/README.md, Tutorials 19 / 26 / 32 / 33 / 37,
    top-level README.md.
  - Source code: src/store/{mod,node}.rs, src/story_view.rs,
    src/tui/{app,keybind,state}.rs.

NOT touched (deliberate):

  - `Book/` — the in-project content book stays intact per
    project policy.
  - `Documentation/RELEASE_NOTES/1.2.4.md` through
    `1.2.7.md` — historical release notes are an
    immutable record of what shipped under what name at
    that time.
  - `Documentation/RELEASE_NOTES/README.md` — release
    index references historical titles by their
    contemporary names.
